# Forum Debate

## What this skill does

This skill turns multi-agent disagreement into a usable decision process.

Instead of collecting isolated answers and stitching them together, it forces a more disciplined flow:
- parallel research
- first-round positions
- disagreement mapping
- bounded debate
- consensus synthesis

The goal is not noise.
The goal is productive collision.

## Why it matters

When multiple agents answer in parallel, the usual failure mode is shallow aggregation.

You get:
- duplicated points
- hidden contradictions
- weak assumptions left unchallenged
- no clear boundary between fact and interpretation

This skill is designed to avoid that.

## Best use cases

Use this skill when you need to:
- compare competing technical directions
- review a product or architecture proposal
- run high-stakes critique before a decision
- generate a consensus memo from multiple expert perspectives

## Core roles

### Research agents

Each agent studies the problem from a distinct angle.

### Moderator

The moderator does not push a preferred answer. The moderator identifies agreement, surfaces contradiction, bounds the debate, and produces the synthesis.

### Reporting agent

The reporting layer converts the conclusion into the required artifact.

## Standard phases

1. Parallel research
2. First-round statements
3. Disagreement mapping
4. Bounded debate
5. Consensus synthesis
6. Final report output

## What good output looks like

A strong outcome should contain:
- final conclusion
- retained disagreements
- action recommendations
- risks
- assumptions that still need validation

## When not to use it

Do not use this skill when:
- the task is simple and factual
- one specialist already owns the answer
- the debate would be fake because no evidence is available
- time pressure makes structured disagreement wasteful
